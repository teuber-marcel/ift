iftRISF.h File Reference

Description

Extension of the ISF interface to perform superpixel segmentation over superpixel graphs.

Author
Felipe Lemes Galvao
Date
July 7, 2017

This file is separated into three parts identified by their main struct: iftISFOptions, iftForestAnnotationInfo and iftSuperpixelIGraph.

The ISFOptions part was originally intended to simplify the access to different versions of the original ISF algorithm based on iftIGraph. It currently serves a similar purpose in the RISF framework by defining how a single segmentation step (i.e. ISF generalized to superpixel graphs) is to be performed over the iftSuperpixelIGraph.

The iftForestAnnotation part is intended to provide standalone data and operations for IFT (i.e. the framework) graphs in general. Still needs some improvements to be worth using in other contexts.

The iftSuperpixelIGraph defines the image graph used in single level of the RISF framework, and the relevant operations to perform superpixel segmentation with ISF (seen as a single segmentation step in framework). It is designed to work primarily over RAGs by accessing a superpixel label map (refImg) to map the image domain to graph nodes.

Note that there is no explicit hierarchy struct, each iftSuperpixelIGraph represents a single level, and adjacent levels are implicitly linked by their refImg.

Use mean of mimage values as superpixel feature.

Author
Felipe Lemes Galvao
Date
June 27, 2017

For each superpixel, takes the mean (average) value of its pixels' feature vectors (i.e. mimg values) as superpixel feature vector.

For now we assume that the number of pixels in each superpixel is small enough so that naive float accumulation does not lead to significant errors.

The returned iftMatrix is owned by the caller.

◆ iftInitSuperpixelIGraph()

iftSuperpixelIGraph * iftInitSuperpixelIGraph ( iftImage *  refImg)

Creates a new superpixel graph.

Author
Felipe Lemes Galvao
Date
July 31, 2017

The created graph still lacks edges and features, which are added through other functions in this header.

Number of nodes is implicitly defined as the maximum value in refImg. Additionally, if refImg has missing labels in between, independent nodes (i.e., no neighbors) might be created later on.

The returned iftSuperpixelIGraph is owned by the caller.

See also
iftDestroySuperpixelIGraph()
