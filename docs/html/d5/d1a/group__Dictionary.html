Dictionary
DataStructures

Description

Dictionary structure, based on Hash Tables, that stores pairs of generic keys and values.

Note
Examples can be found in demo/Miscellaneous/DataStructures/iftTestDict*.c

The allowed Datatypes for the keys are: IFT_CHAR_TYPE, IFT_UCHAR_TYPE, IFT_STR_TYPE, SHORT_TYPE, INT_TYPE, IFT_LONG_TYPE, USHORT_TYPE, INT_TYPE, IFT_LONG_TYPE, FLOAT_TYPE, IFT_DBL_TYPE, IFT_PTR_TYPE.
The allowed Datatypes for the value are the same of the key plus IFT_PTR_TYPE
. Initially, empty buckets of the dictionary has key type and val type equals to IFT_UNTYPED.

  KEY_INDEX 
)    iftDictContainGKey(iftCreateGVal((KEY)), (DICT), (KEY_INDEX))

Checks if the dict contains the key. SUPER SLOW FUNCTION IF USED INSIDE LOOPS.

Author
Samuel Martins
Date
Feb 16, 2016

When seeking the key, it ignores empty buckets and continue the search until all buckets have been visited.
This enables that a key B rehashed (i.e., assigned to a new position), due to a colision with a key A, is able to reach its real position even if the key A have been removed.

Parameters
KEYThe key to be checked.
DICTThe considered dict.
KEY_INDEXReturns the dict table index of the key, if it exists. Otherwise, the IFT_HASH_FAIL is returned.
Returns
True if the dict contains the key and false otherwise.
Attention
If the key_index is NULL, nothing is returned to it.
Warning
If the GVal key is indeed a string, it will be deallocated at the end of the function.

◆ iftCreateDictWithApproxSize()

iftDict * iftCreateDictWithApproxSize ( size_t  n)

Creates a Dictionary with an approximate size.

Author
Samuel Martins
Date
Jan 17th, 2016

Initially, empty buckets of the dictionary has key type and val type equals to IFT_UNTYPED.

Actually, the real dictionary size is computed from the passed approximate size n, and it is a bigger and better size used to minimize collisions due to the hashing.
The real dictionary size is found according to the following rules:

  1. size >= n;
  2. size is the farthest prime number of the powers of 2 that surround n;
  3. size and (1 + (k % m)), where m = size - 2, do not have GCD (greatest common divisor);
  4. (1 + (k % m)) > size, where m = size - 2.
Parameters
nThe approximate size of the Dict.
Returns
The created dictionary.
Note
When the dict is FULL, it will be resized automatically.
Initially, empty buckets of the dictionary has key type and val type equals to IFT_UNTYPED.
See also
iftCreateDict(), iftCreateDictWithInitialSize()
